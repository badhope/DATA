# Creative Writing Assistant Skill

创意写作与特殊场景内容生成的完整指南。

## 内容类型体系

### 1. 营销文案

| 类型 | 目的 | 特点 |
|------|------|------|
| 广告文案 | 吸引注意 | 简短有力、行动导向 |
| 软文 | 植入品牌 | 故事性强、软性植入 |
| 产品描述 | 销售转化 | 突出卖点、营造紧迫 |
| 邮件营销 | 激活用户 | 个性化、价值导向 |

### 2. 社交媒体内容

| 平台 | 内容特点 | 最佳实践 |
|------|----------|----------|
| 微博 | 简短、话题性强 | 配图、使用热门话题 |
| 微信公众号 | 深度、传播性 | 标题吸引人、干货足 |
| 小红书 | 种草、生活方式 | 图片精美、真实感 |
| 抖音/视频号 | 短平快、娱乐性 | 前3秒抓眼球 |

### 3. 故事创作

| 类型 | 结构 | 适用场景 |
|------|------|----------|
| 短篇 | 起承转合 | 社交媒体、邮件 |
| 中篇 | 多章节 | 博客、小说 |
| 连载 | 悬念钩子 | 持续吸引 |

## 营销文案框架

### AIDA 模型

```
Attention (注意) ──> Interest (兴趣) ──> Desire (欲望) ──> Action (行动)
     │                   │                  │                │
     ▼                   ▼                  ▼                ▼
   吸引眼球的         展示价值            激发情感          明确号召
   开头文案           特色介绍            案例背书          购买按钮
```

### PAS 框架

```markdown
## Problem-Agitation-Solution

### Problem (问题)
你是否遇到过...

### Agitation (放大)
想象一下，每天...

### Solution (解决方案)
现在，有了[产品名]...

---

## 示例: 在线课程销售

**Problem**: 想学编程但不知道从何开始？

**Agitation**: 网上教程太零散，学了后面忘前面。花费大量时间却收效甚微，还可能学到错误的内容...

**Solution**: Python入门实战班 - 8周从零基础到独立项目。专业导师带路，社群答疑，毕业项目实战。

👉 扫码立即报名，前100名送价值299的学习资料包
```

### 促销文案模板

```markdown
## 限时优惠文案模板

### 开头 - 制造紧迫
🚀 重要通知：[活动/产品]即将[变化]

### 中间 - 阐述价值

### 结尾 - 明确行动
⏰ 优惠截止：[时间]
👉 [立即购买/扫码]
💰 更有[额外福利]等你拿
```

## 社交媒体内容策略

### 微博推文结构

```
[开头 - 抓眼球] (1-2行)
│
[正文 - 价值传递] (3-5行)
│
[互动引导 - 提升互动] (1行)
│
[话题标签 - 增加曝光] (1-2个)
```

**示例**:
```
😱 程序员最怕听到的一句话不是"需求变了"

而是...

"这段代码谁写的？"

你遇到过吗？👇

#程序员日常 #代码人生
```

### 小红书种草文案

```markdown
## 种草文案结构

### 封面标题
[数字/痛点] + [解决方案] + [效果保证]

示例：
"3步教你写出爆款文案｜附模板"

### 正文结构
1. **场景引入** (引起共鸣)
   "作为一个每天要和文字打交道的人..."

2. **痛点共鸣** (用户痛点)
   "最头疼的就是..."

3. **解决方案** (产品/方法)
   "后来我发现了这个..."

4. **使用效果** (展示成果)
   "用了之后..."

5. **行动引导** (转化)
   "想要同款？点击左下角..."

### 标签策略
#标签1 #标签2 #标签3 #种草 #好物推荐
```

## 故事创作框架

### 三幕式结构

```markdown
## 第一幕：建置 (25%)
- 介绍主角
- 展示日常生活
- 出现激励事件
- 主角接受挑战

## 第二幕：对抗 (50%)
- 尝试解决问题
- 遇到阻碍
- 最大危机
- 重新评估策略

## 第三幕：解决 (25%)
- 最终决战
- 解决核心问题
- 展示变化成长
- 结局

---

## 短故事示例

### 标题：第37次

"第37次了。"

李明盯着屏幕上"连接失败"四个字，手指悬在键盘上方。

他还记得三年前第一次连接成功的兴奋。那时候，整个实验室都在欢呼。他以为自己解决了困扰人类十年的难题。

直到他发现了那个bug。

修复一个bug需要三十七天。而每修复一个，就会产生两个新的。

"还要继续吗？"助手的声音从身后传来。

李明没有回答。他开始输入第38段代码。

---
字数：约200字
主题：科技与人性
结构：起承转合完整
```

## 内容风格指南

### 正式商务风格

- 语言简洁专业
- 避免口语化表达
- 结构清晰分层
- 数据支撑观点

### 轻松亲和风格

- 适当使用口语
- 增添幽默元素
- 拉近与读者距离
- 营造亲切感

### 技术科普风格

- 复杂概念简化
- 使用类比解释
- 图文配合
- 循序渐进
