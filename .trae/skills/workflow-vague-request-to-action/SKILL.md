# Vague Request to Action Workflow

将模糊需求转化为明确行动计划的工作流。

## When to Use

- 用户需求不明确
- 需要澄清目标和范围
- 模糊的问题描述
- 初始需求分析

## Workflow Steps

1. **理解原始需求** - 记录用户的原始描述
2. **澄清关键问题** - 提问以明确不确定性
3. **定义目标** - 将需求转化为具体目标
4. **分解任务** - 划分可执行的步骤
5. **确认方案** - 与用户确认执行计划

## Related Prompts

- `prompt-meta-refine-ambiguous-request` - 细化模糊请求
- `prompt-task-planning-create-execution-plan` - 创建执行计划
