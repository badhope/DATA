# Repo Reading to Change Plan Workflow

理解代码后制定修改方案的工作流。

## When to Use

- 需要修改现有代码
- 理解代码结构和逻辑
- 制定变更计划
- 评估修改影响

## Workflow Steps

1. **分析仓库结构** - 了解项目组织
2. **定位目标代码** - 找到需要修改的文件
3. **理解代码逻辑** - 分析依赖和调用关系
4. **制定变更计划** - 规划修改步骤和风险
5. **评估影响** - 检查对其他模块的影响
