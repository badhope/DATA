# Tool Assisted Debug Workflow

利用工具系统性地调试问题的工作流。

## When to Use

- 需要系统性调试
- 使用多个调试工具
- 复杂问题需要工具辅助
- 日志和断点调试

## Workflow Steps

1. **分析问题** - 确定调试目标和范围
2. **选择工具** - 根据问题类型选择合适工具
3. **收集信息** - 使用工具获取运行时信息
4. **分析数据** - 解读工具输出
5. **定位问题** - 确定根本原因
6. **验证修复** - 确认问题解决
