# Context

声明需要证据支持。本 prompt 的核心目标是：**指导 AI 用可用证据系统性地验证声明，评估声明的可信度**。

# Prompt Body

## 阶段 1：声明分析

### 1.1 声明类型

```markdown
## 声明类型

### 事实性声明
```markdown
**定义**: 可以被证实或证伪的陈述
**示例**: "Python 3.12发布于2023年10月"
**验证方式**: 追溯原始来源
```

### 观点性声明
```markdown
**定义**: 表达意见或判断的陈述
**示例**: "React是最好的前端框架"
**验证方式**: 评估论据和支持
```

### 推断性声明
```markdown
**定义**: 基于证据得出的结论
**示例**: "这个方案会导致性能问题"
**验证方式**: 验证推理逻辑和前提
```

### 预测性声明
```markdown
**定义**: 关于未来事件的陈述
**示例**: "这个系统明年会处理10倍流量"
**验证方式**: 评估假设和模型
```
```

### 1.2 声明结构

```markdown
## 声明结构

### 声明要素
```markdown
**声明**: [声明内容]

**声明类型**: [类型]
**核心断言**: [核心内容]
**隐含假设**: [假设]
**支持需求**: [需要什么支持]
```
```

## 阶段 2：证据评估

### 2.1 证据类型

```markdown
## 证据类型

### 证据强度排序
| # | 类型 | 强度 | 说明 |
|---|------|------|------|
| 1 | 直接证据 | 最强 | 直接证明声明的事实 |
| 2 | 间接证据 | 强 | 通过推理支持声明 |
| 3 | 专家意见 | 中 | 专家的支持性观点 |
| 4 | 案例证据 | 弱-中 | 具体案例的支持 |
| 5 | 类比 | 弱 | 通过相似情况推断 |
| 6 | 权威声明 | 可变 | 取决于权威来源的可靠性 |

### 证据质量评估
```markdown
**高质量证据特征**:
- 可重复验证
- 来源可靠
- 与声明直接相关
- 无明显偏差

**低质量证据特征**:
- 难以验证
- 来源不可靠
- 与声明间接相关
- 存在偏差
```

## 阶段 3：验证执行

### 3.1 验证方法

```markdown
## 验证方法

### 直接验证
```markdown
**适用**: 事实性声明
**方法**: 追溯原始来源
**检查**:
1. 来源是否可靠？
2. 陈述是否准确？
3. 是否有其他来源确认？
```

### 逻辑验证
```markdown
**适用**: 推断性声明
**方法**: 检验推理逻辑
**检查**:
1. 前提是否正确？
2. 推理是否有效？
3. 结论是否必然？
```

### 一致性验证
```markdown
**适用**: 观点性声明
**方法**: 检查与其他证据的一致性
**检查**:
1. 是否与已知事实一致？
2. 是否与其他可靠来源一致？
3. 是否内部一致？
```
```

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `claim` | 声明 | `"Python性能比JavaScript好"` |
| `evidence_available` | 可用证据 | `[{source: "...", content: "..."}]` |
| `claim_category` | 声明类别 | `"观点性"` |

# Usage Notes

1. **证据说话**：声明的可信度由证据决定
2. **透明评估**：清楚说明证据的强度和局限性
3. **区分类型**：不同类型的声明用不同的验证方法
4. **不过度推断**：不从有限证据过度推断
5. **承认局限**：当证据不足时承认不确定性

# Example Input

```yaml
claim: "React是2023年最流行的前端框架"
evidence_available:
  - source: "GitHub Stars"
    content: "React有21万stars"
  - source: "npm下载量"
    content: "React周下载量1000万"
claim_category: "事实性+观点性"
```

# Example Output

```yaml
verification_result:
  result: "部分支持"
  confidence_adjustment: "downgrade to medium"

evidence_strength: "medium"

claim_reliability: |
  "React是2023年最流行的前端框架"这个声明是部分支持的。

  支持部分：
  - GitHub stars高（21万）
  - npm下载量大（周1000万）

  质疑部分：
  - "最流行"定义不明确
  - 未比较其他框架如Vue、Angular
  - 缺乏综合排名数据

transparent_report: |
  **声明**: React是2023年最流行的前端框架

  **验证结果**: 部分支持

  **证据**:
  - React确实有很高的使用量
  - 但缺乏与其他框架的系统性比较

  **置信度**: 中等

  **建议**: 如需更准确结论，需要更多框架的对比数据
```
