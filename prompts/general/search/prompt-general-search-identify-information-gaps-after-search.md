# Context

搜索不一定能回答所有问题。识别信息缺口并坦诚告知用户是专业的行为。本 prompt 的核心目标是：**指导 AI 在搜索完成后系统性地识别信息缺口，并给出下一步建议**。

# Prompt Body

## 阶段 1：缺口识别

### 1.1 缺口类型

```markdown
## 缺口类型

### 事实缺口
```markdown
**定义**: 缺乏具体的事实或数据
**示例**: "具体数字未知"、"确切日期不确定"
**影响**: 无法给出精确答案
**缓解**: 给出范围估计或说明不确定性
```

### 上下文缺口
```markdown
**定义**: 缺乏具体场景或背景信息
**示例**: "需要知道团队规模"、"需要知道项目阶段"
**影响**: 无法给出针对性建议
**缓解**: 给出通用建议 + 场景化建议
```

### 来源缺口
```markdown
**定义**: 缺乏权威来源支持
**示例**: "只有非官方来源"
**影响**: 结论可靠性降低
**缓解**: 明确标注来源可靠性
```

### 时效性缺口
```markdown
**定义**: 信息可能已经过时
**示例**: "最新信息无法确认"
**影响**: 结论可能不适用于当前
**缓解**: 说明信息时效性
```

### 完整性缺口
```markdown
**定义**: 答案不完整，有遗漏
**示例**: "这是部分解决方案"
**影响**: 可能误导用户
**缓解**: 明确说明这是部分答案
```
```

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `search_results` | 搜索结果 | `[{source: "...", content: "..."}]` |
| `original_question` | 原始问题 | `"应该用什么架构"` |
| `answer_provided` | 已提供的答案 | `"建议微服务架构"` |

# Usage Notes

1. **不假装完美**：承认信息不足比假装完美更专业
2. **评估影响**：评估每个缺口对结论的影响
3. **提供缓解**：给出缓解缺口的建议
4. **明确下一步**：给出明确的进一步行动建议
5. **透明沟通**：清楚地向用户说明缺口情况

# Example Input

```yaml
search_results:
  - source: "技术博客"
    content: "微服务架构的实践经验"
original_question: "我们应该选择微服务还是单体架构"
answer_provided: "根据搜索结果，微服务架构适合大型团队..."
```
