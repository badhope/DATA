# Context

不是所有问题都需要搜索。正确的搜索决策需要平衡准确性和效率。本 prompt 的核心目标是：**指导 AI 建立系统性的搜索决策框架，判断何时需要搜索**。

# Prompt Body

## 阶段 1：决策框架

### 1.1 搜索必要性维度

```markdown
## 搜索必要性维度

### 必要性评估维度
| # | 维度 | 说明 | 高必要性特征 |
|---|------|------|--------------|
| 1 | 时效性 | 信息是否容易过时 | 最新版本、新发布、近期变化 |
| 2 | 准确性要求 | 错误的代价 | 决策支持、技术实现、精确数据 |
| 3 | 确定性 | 当前知识的确定程度 | 不确定、模糊、需要验证 |
| 4 | 覆盖度 | 当前知识是否充分 | 知识不足、领域不熟悉 |
| 5 | 可验证性 | 是否容易验证 | 难以通过推理验证 |
```

### 维度权重
```markdown
**默认权重**:
- 时效性: 30%
- 准确性要求: 25%
- 确定性: 20%
- 覆盖度: 15%
- 可验证性: 10%

**可调整**: 根据具体场景调整
```
```

## 阶段 4：决策生成

### 4.1 决策规则

```markdown
## 决策规则

### 明确搜索规则
```markdown
**必须搜索**:
- 时效敏感 + 准确性要求高
- 知识高度不确定 + 准确性要求高
- 版本/发布信息

**建议搜索**:
- 时效敏感 OR 准确性要求高
- 知识不确定 + 可以验证
- 重要决策前的确认

**可选搜索**:
- 时效不敏感 + 准确性要求中
- 可以通过推理解决
- 效率优先场景

**不需要搜索**:
- 时效不敏感 + 准确性要求低
- 知识确定 + 领域熟悉
- 简单确认
```

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `question` | 问题 | `"Python最新版本是多少"` |
| `context` | 上下文 | `{domain: "编程", urgency: "medium"}` |
| `knowledge_status` | 知识状态 | `{certainty: "low", recency: "unknown"}` |

# Usage Notes

1. **系统性决策**：有明确的决策框架
2. **平衡效率**：不是所有问题都需要搜索
3. **明确理由**：给出明确的搜索/不搜索理由
4. **替代方案**：不搜索时提供替代方案
5. **透明置信度**：说明决策的置信度

# Example Input

```yaml
question: "Django 5.0什么时候发布"
context:
  domain: "Python Web开发"
  urgency: "medium"
knowledge_status:
  certainty: "low"
  recency: "unknown"
```

# Example Output

```yaml
search_decision: "必须搜索"

  - "版本发布时间是精确事实，必须准确"

alternative_approaches: []

expected_benefit: "获得准确的Django 5.0发布时间"
