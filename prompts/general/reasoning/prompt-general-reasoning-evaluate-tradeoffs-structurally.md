# Context

任何决策都涉及权衡。结构化的权衡分析能帮助决策者理解选择的代价和收益。本 prompt 的核心目标是：**指导 AI 结构化地评估权衡，让决策者清楚每种选择的利弊**。

# Prompt Body

## 阶段 1：权衡结构定义

### 1.1 权衡维度

```markdown
## 权衡维度

### 核心维度
| # | 维度 | 说明 | 权衡方向 |
|---|------|------|----------|
| 1 | 成本 | 投入的资源 | 越低越好/越高越好 |
| 2 | 收益 | 获得的收益 | 越高越好 |
| 3 | 风险 | 潜在的风险 | 越低越好 |
| 4 | 时间 | 所需时间 | 越短越好 |
| 5 | 质量 | 质量水平 | 越高越好 |
| 6 | 灵活性 | 适应变化能力 | 越高越好 |

### 维度权衡关系
```markdown
**典型权衡关系**:
- 成本 ↔ 时间: 投入更多资源可以缩短时间
- 成本 ↔ 质量: 投入更多可以提高质量
- 灵活性 ↔ 简单性: 更灵活通常意味着更复杂
- 速度 ↔ 风险: 快速方案通常风险更高
```
```

### 1.2 权衡类型

```markdown
## 权衡类型

### 直接权衡
```markdown
**定义**: 一个维度增加导致另一个维度减少
**示例**: 更多功能 vs 更短时间
**分析方式**: 量化各方变化，计算净收益
```

### 间接权衡
```markdown
**定义**: 改变一个维度会影响多个其他维度
**示例**: 选择新技术（影响成本、时间、风险）
**分析方式**: 分解影响，跟踪连锁反应
```

### 长期/短期权衡
```markdown
**定义**: 短期成本换取长期收益
**示例**: 技术债务 vs 快速交付
**分析方式**: 全生命周期成本分析
```
```

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `decision_options` | 决策选项 | `[{name: "A", description: "..."}]` |
| `evaluation_dimensions` | 评估维度 | `["成本", "收益", "风险"]` |
| `stakeholder_perspectives` | 利益相关方视角 | `[{role: "技术团队", concerns: ["..."]}]` |

# Usage Notes

1. **全面权衡**：分析所有重要维度，不遗漏关键权衡
2. **量化分析**：尽可能量化权衡，便于比较
3. **透明关系**：明确权衡关系，不是简单的优劣对比
4. **多视角**：考虑不同利益相关方的视角
5. **敏感性分析**：分析关键假设变化对结论的影响

# Example Input

```yaml
decision_options:
  - name: "自研方案"
    description: "自己开发系统"
  - name: "采购方案"
    description: "购买成熟产品"
evaluation_dimensions:
  - "成本"
  - "时间"
  - "可控性"
  - "维护性"
stakeholder_perspectives:
  - role: "技术团队"
    concerns: ["开发工作量大"]
  - role: "业务方"
    concerns: ["交付时间"]
```
