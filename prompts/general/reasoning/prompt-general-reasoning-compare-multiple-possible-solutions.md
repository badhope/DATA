# Context

在面对多个可行方案时，需要系统化的比较方法而非凭感觉选择。本 prompt 的核心目标是：**指导 AI 在多个方案之间进行结构化的多维度比较，得出有据可依的推荐结论**。

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `problem` | 问题 | `"选择什么架构方案"` |
| `solutions` | 方案列表 | `[{name: "A", description: "..."}]` |
| `criteria` | 评估标准 | `["技术可行性", "成本"]` |

# Usage Notes

1. **维度全面**：覆盖技术、商业、风险等多个维度
2. **权重合理**：根据问题特点设置权重
3. **评分客观**：基于事实评分，避免主观偏好
4. **推荐明确**：给出明确的推荐和理由
5. **不确定性透明**：说明评估的不确定性

# Example Input

```yaml
problem: "用户量预计增长10倍，需要升级架构"
solutions:
  - name: "单体优化"
    description: "优化现有单体架构"
  - name: "水平扩展"
    description: "无状态改造+负载均衡"
  - name: "微服务"
    description: "拆分为微服务"
criteria:
  - "技术可行性"
  - "成本"
  - "风险"
  - "可维护性"
```
