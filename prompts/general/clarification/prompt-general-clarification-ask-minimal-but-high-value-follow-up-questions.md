# Context

不是所有问题都需要问。好的跟进问题应该精准且有价值。本 prompt 的核心目标是：**指导 AI 设计最少但最高价值的问题组合，在最小化用户负担的同时最大化理解清晰度**。

# Prompt Body

## 阶段 1：不确定性分析

### 1.1 不确定性来源

```markdown
## 不确定性来源

### 任务相关
```markdown
**目标不确定性**: 对用户真正目标的理解
**范围不确定性**: 任务应该包含什么
**标准不确定性**: 怎样算成功
**约束不确定性**: 有什么限制
```

### 上下文相关
```markdown
**环境不确定性**: 系统/技术环境
**用户相关不确定性**: 用户背景/偏好
**资源不确定性**: 时间/成本/人力
```

### 方案相关
```markdown
**方案不确定性**: 方案选择
**实现不确定性**: 如何实现
**风险不确定性**: 潜在风险
```
```

### 1.2 不确定性评估

```markdown
## 不确定性评估

### 剩余不确定性
| # | 不确定性 | 对任务的影响 | 解决价值 | 评估 |
|---|----------|--------------|----------|------|
| 1 | [不确定] | [影响] | [价值] | [高/中/低] |
```

### 关键不确定性
```markdown
**最需要解决的不确定性**:
1. [不确定1] - 影响: [影响] - 价值: [价值]
2. [不确定2] - 影响: [影响] - 价值: [价值]
```
```

## 阶段 2：问题设计

### 2.1 问题价值评估

```markdown
## 问题价值评估

### 评估维度
| # | 维度 | 说明 | 权重 |
|---|------|------|------|
| 1 | 信息价值 | 能解决多少不确定性 | 40% |
| 2 | 推断能力 | 能间接推断多少其他信息 | 30% |
| 3 | 回答成本 | 用户需要多少时间回答 | 20% |
| 4 | 风险降低 | 能降低多少执行风险 | 10% |

### 价值计算
```markdown
**问题**: [问题]

**信息价值**: [高/中/低]
**推断能力**: [高/中/低]
**回答成本**: [低/中/高]
**风险降低**: [高/中/低]

**综合价值**: [评分]
```
```

### 2.2 问题候选

```markdown
## 问题候选

### 高价值问题
| # | 问题 | 信息价值 | 推断能力 | 回答成本 | 综合 |
|---|------|----------|----------|----------|------|
| 1 | [问题] | [高] | [高] | [低] | [高] |
```

### 低价值问题
```markdown
**应该避免的问题**:
- [问题1] - 理由: [理由]
- [问题2] - 理由: [理由]
```
```

## 阶段 3：问题组合优化

### 3.1 组合分析

```markdown
## 组合分析

### 问题之间的关系
```markdown
**独立问题**: 可以单独问，不影响其他问题的价值
**依赖问题**: 问了一个之后另一个才相关
**重叠问题**: 问了一个就能推断另一个
**矛盾问题**: 两个问题可能得到矛盾答案
```

### 优化目标
```markdown
**目标**: 用最少的问题解决最多的不确定性

**约束**:
- 问题数量 ≤ [最大数量]
- 回答时间 ≤ [最大时间]
- 覆盖所有高价值不确定性
```
```

### 3.2 最优组合

```markdown
## 最优组合

### 推荐组合
```markdown
**问题组合**: [组合描述]

**包含的问题**:
1. [问题1] - 解决: [不确定性]
2. [问题2] - 解决: [不确定性]

**为什么这个组合最优**:
- 覆盖: [覆盖的不确定性]
- 推断: [可推断的其他信息]
- 回答成本: [总成本]
```

### 备选组合
```markdown
**如果推荐组合不可行**:

**最小组合** (只需1个答案):
- [问题]

**扩展组合** (如果需要更清晰):
- [问题1]
- [问题2]
```
```

## 阶段 4：问题输出

### 4.1 精准问题格式

```markdown
## 精准跟进问题

### 核心问题
```markdown
**[问题1]**

**为什么问这个**: [原因]

**帮助我理解**: [理解什么]

**回答格式建议**: [格式]
```

### 可选问题
```markdown
**如果可以的话**:
[可选问题]
- 背景: [为什么可能有用]
```
```

### 4.2 问题呈现

```markdown
## 需要了解的

**为了更好地帮助您，我需要了解**:

**[最重要的问题]**

[具体问题]

---

**[可选] 如果您能补充**:

[次要问题]
（这个不是必需的，但能帮助我更好地理解）
```
```

## 阶段 5：自适应策略

### 5.1 回答驱动的下一步

```markdown
## 回答后的下一步

### 如果回答了问题A
```markdown
**根据回答**:
- 如果 [答案X] → 继续执行 [方案1]
- 如果 [答案Y] → 需要跟进 [问题B]
```

### 决策树
```markdown
**问题A** → [答案]
     ↓
[分支决策]
```
```

### 5.2 快速收敛策略

```markdown
## 快速收敛

### 如果时间紧迫
```markdown
**只需回答1个问题**:
[最关键的问题]

**回答后**: 可以立即开始执行
```

### 如果需要完整理解
```markdown
**请再补充**:
[次关键的问题]

**回答后**: 可以完整规划
```
```
