# Context

宽泛的任务（如"帮我写个系统"）需要先收窄范围才能有效执行。本 prompt 的核心目标是：**指导 AI 帮助用户识别和收窄任务范围，明确执行的边界和重点**。

# Prompt Body

## 阶段 1：范围分析

### 1.1 范围分解

```markdown
## 范围分解

### 任务分解
```markdown
**原始任务**: [任务]

**可能包含的工作**:
| # | 工作项 | 估计工作量 | 优先级 | 说明 |
|---|--------|------------|--------|------|
| 1 | [项1] | [量] | [高/中/低] | [说明] |
| 2 | [项2] | [量] | [高/中/低] | [说明] |
| 3 | [项3] | [量] | [高/中/低] | [说明] |
```

### 范围维度
```markdown
**功能维度**: 包含哪些功能
**质量维度**: 达到什么质量标准
**技术维度**: 涉及哪些技术栈
**时间维度**: 需要多长时间
```
```

### 1.2 范围边界

```markdown
## 范围边界

### 边界识别
```markdown
**范围内**:
- [项1]
- [项2]

**范围外**（需要明确排除）:
- [项1]
- [项2]
```

### 边界模糊区
```markdown
**可能范围内也可能范围外**:
- [项] - 需要确认
```
```

## 阶段 2：收窄策略

### 2.1 收窄维度

```markdown
## 收窄维度

### 功能收窄
```markdown
**策略**: 聚焦核心功能 vs 包含完整功能

**选项**:
1. 核心功能优先（80%价值在20%功能）
2. 完整功能集
3. MVP版本

**建议**: [基于情况的选择]
```

### 质量收窄
```markdown
**策略**: 高质量 vs 快速交付

**选项**:
1. 最高质量
2. 生产就绪
3. 原型/演示

**建议**: [基于情况的选择]
```

### 2.2 价值分析

```markdown
## 价值分析

### 价值-工作量矩阵
| # | 工作项 | 价值 | 工作量 | 价值/工作量比 |
|---|--------|------|--------|----------------|
| 1 | [项] | [高/中/低] | [大/中/小] | [比值] |
```

### 优先工作项
```markdown
**高价值低工作量**: [项] - 优先做
**高价值高工作量**: [项] - 重点做
**低价值低工作量**: [项] - 考虑做
**低价值高工作量**: [项] - 跳过
```
```
