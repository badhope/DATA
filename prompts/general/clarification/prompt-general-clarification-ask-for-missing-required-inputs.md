# Context

完成任务需要必要的输入。缺失输入时继续执行往往会导致错误或返工。本 prompt 的核心目标是：**指导 AI 系统性地识别缺失的必要输入，主动向用户询问**。

# Prompt Body

## 阶段 1：输入分析

### 1.1 输入类型

```markdown
## 输入类型

### 必要输入
```markdown
**定义**: 没有就无法执行或会严重错误的输入
**特征**: 必须在执行前获取
**示例**: 要修改的文件、目标环境、关键参数
**处理**: 必须询问获取
```

### 可选输入
```markdown
**定义**: 可以有默认值或不需要的输入
**特征**: 没有也可以执行
**示例**: 注释风格、超时设置
**处理**: 可以使用默认值或跳过
```

### 条件输入
```markdown
**定义**: 特定条件下才需要的输入
**特征**: 根据情况判断是否需要
**示例**: 特定配置、认证信息
**处理**: 根据场景判断是否询问
```
```

### 1.2 输入识别

```markdown
## 输入识别

### 任务分解与输入需求
```markdown
**任务**: [任务]

**分解的步骤**:
1. [步骤1] → 需要: [输入]
2. [步骤2] → 需要: [输入]
3. [步骤3] → 需要: [输入]
```

### 输入清单
| # | 输入 | 类型 | 必要性 | 可用性 | 缺失影响 |
|---|------|------|--------|--------|----------|
| 1 | [输入] | [类型] | [必要/可选/条件] | [✅/❌] | [影响] |
```

|---|----------|------|--------|----------|----------|

```markdown

### 阻断性问题
```markdown

```markdown

3. **有格式**: 说明期望的格式
4. **有示例**: 帮助理解的示例
5. **可回答**: 用户能够回答
