# Context

用户经常表达模糊的想法（如"想做一个很酷的功能"），需要转化为清晰的任务定义才能执行。本 prompt 的核心目标是：**指导 AI 将用户模糊的想法通过澄清问题转化为清晰、可执行的任务定义**。

# Prompt Body

## 阶段 1：想法分析

### 1.1 想法拆解

```markdown
## 想法拆解

### 从模糊想法中提取要素
```markdown
**原始想法**: [想法]

**提取的要素**:
| # | 要素 | 提取方式 | 说明 |
|---|------|----------|------|
| 1 | [要素] | [从想法中提取] | [说明] |
```

### 要素类型
```markdown
**想要的成果**: 用户想达到什么效果
**约束条件**: 用户提到了什么限制
**问题背景**: 用户为什么有这个想法
**大致方向**: 用户倾向什么样的方案
**期望价值**: 用户期望获得什么价值
```
```

### 1.2 理解验证

```markdown
## 理解验证

### 我的理解
```markdown
**您的想法是**: [我的理解]

**您想要**: [成果]

**您关心**: [价值]

**是这样吗**? [是/不完全是/不是]
```
```

## 阶段 2：澄清问题

### 2.1 关键问题

```markdown
## 关键澄清问题

### 问题清单
| # | 问题 | 目的 | 优先级 |
|---|------|------|--------|
| 1 | [问题] | [目的] | [高/中/低] |
```

### 问题设计
```markdown
### 关于成果
**问题**: "您希望这个功能达到什么效果？"
**为什么问**: 明确目标

### 关于用户
**问题**: "这个是给谁用的？"
**为什么问**: 了解受众

### 关于现有系统
**问题**: "是在现有系统上改进还是新建？"
**为什么问**: 确定上下文

### 关于约束
**问题**: "有什么限制吗？比如时间、技术、成本？"
**为什么问**: 了解约束
```

### 2.2 回答收集

```markdown
## 回答收集

### 已收集的回答
| # | 问题 | 回答 | 对理解的影响 |
|---|------|------|--------------|
| 1 | [问题] | [回答] | [影响] |
```

## 阶段 3：任务定义

### 3.1 任务要素

```markdown
## 任务要素定义

### 目标
```markdown
**任务目标**: [清晰定义的目标]

**目标特征**:
- 具体: [具体描述]
- 可衡量: [衡量标准]
- 可达成: [是否可行]
- 相关: [与用户需求的相关性]
- 有时限: [时间要求]
```

### 范围
```markdown
**包含**:
1. [项1]
2. [项2]

**不包含**:
1. [项1]
2. [项2]
```

### 约束
```markdown
**技术约束**: [约束]
**时间约束**: [约束]
**资源约束**: [约束]
**其他约束**: [约束]
```
```

### 3.2 任务定义

```markdown
## 任务定义

### 任务名称
[清晰的任务名称]

### 任务描述
```markdown
[清晰描述要做什么]

**目标**: [目标]

**范围**: [范围]

**约束**: [约束]
```

### 任务产出
```markdown
**交付物**:
1. [交付物1]
2. [交付物2]

**质量标准**: [标准]
```
```

## 阶段 4：成功标准

### 4.1 成功标准定义

```markdown
## 成功标准

### 功能标准
```markdown
**完成定义**: [什么算完成]

**验收标准**:
1. [标准1]
2. [标准2]
```

### 质量标准
```markdown
**性能要求**:
- [要求]

**质量要求**:
- [要求]
```

### 4.2 成功标准确认

```markdown
## 成功标准确认

### 标准清单
| # | 标准 | 优先级 | 可衡量 |
|---|------|--------|--------|
| 1 | [标准] | [高/中/低] | [✅/❌] |

### 确认问题
```markdown
**这些标准是否完整反映了您的期望**?
- 如有补充: [请补充]
- 如有修改: [请修改]
```
```

## 阶段 5：输出模板

### 5.1 完整转化模板

```markdown
## 任务定义

### 原始想法
```markdown
[用户的模糊想法]
```

### 任务定义
```markdown
**任务名称**: [名称]

**任务描述**:
[清晰描述]

**目标**:
1. [目标1]
2. [目标2]

**范围**:
- 包含: [列表]
- 不包含: [列表]

**约束**:
- [约束]

**交付物**:
1. [物1]
2. [物2]
```

### 成功标准
```markdown
**成功标准**:
1. [标准1]
2. [标准2]

**质量标准**:
- [标准]
```

### 请确认
```markdown
**这个任务定义是否符合您的想法**?
- 如有调整: [请告知]
```
```

```markdown
