# Context

当任务被中断后重新继续时，AI 需要系统性地恢复相关上下文。这不是"回忆"（因为 AI 没有真正的记忆），而是在可见信息中重建上下文状态。本 prompt 的核心目标是：**指导 AI 在继续任务前正确恢复相关上下文，避免因上下文丢失导致的质量问题**。

# Prompt Body

## 阶段 1：中断点识别

### 1.1 中断类型

```markdown
## 中断类型

### 中断原因
| # | 类型 | 说明 | 上下文保留程度 |
|---|------|------|----------------|
| 1 | 对话间隔 | 用户离开后返回 | 取决于历史记录 |
| 2 | 任务切换 | 临时处理其他事 | 取决于记录完整性 |
| 3 | 系统重置 | 对话上下文丢失 | 需要完全重建 |
| 4 | 明确中断 | 用户要求暂停 | 取决于中断记录 |

### 中断影响评估
```markdown
**中断时间**: [时间]
**中断长度**: [长度]
**可能影响**:
1. [影响1]
2. [影响2]
```
```

**时间戳**: [时间]
```

| # | 上下文项 | 最后状态 | 确认必要性 |
|---|----------|----------|------------|

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `current_task` | 当前任务 | `"继续重构用户服务"` |
| `previous_context` | 之前上下文 | `"之前在讨论API设计"` |
| `task_interruption` | 任务是否被中断 | `true` |

# Usage Notes

1. **不假设完整性**：不要假设上下文是完整的
2. **主动确认**：主动询问用户确认关键上下文
3. **标注缺口**：对于无法恢复的缺口，明确标注
4. **说明恢复状态**：继续任务前说明当前状态
5. **降低风险**：对于低置信度项，采取预防措施

# Example Input

```yaml
current_task: "继续用户登录功能实现"
previous_context: "之前完成了登录API设计，需要继续实现"
task_interruption: true
```
