# Context

在迭代过程中会产生大量的决策，这些决策需要有结构地记录下来，以便后续回溯和理解。决策历史不只是记录"做了什么"，更重要的是记录"为什么这么做"和"当时的情境是什么"。本 prompt 的核心目标是：**指导 AI 系统性地记录决策历史，支持决策回溯和决策理解**。

# Prompt Body

## 阶段 1：决策识别

### 1.1 决策类型

```markdown
## 决策类型

### 架构决策
```markdown
**定义**: 影响系统架构的决策
**特征**: 高影响、难以逆转
**示例**:
- 选择微服务架构
- 选择数据库类型
- 选择认证方案

**记录重点**:
- 决策内容
- 技术依据
- 替代方案
- 影响范围
```

### 设计决策
```markdown
**定义**: 影响代码设计的决策
**特征**: 中等影响、可重构
**示例**:
- 模块划分方式
- 接口设计
- 设计模式选择

**记录重点**:
- 决策内容
- 设计原则
- 权衡考量
```

### 实现决策
```markdown
**定义**: 具体的实现方式决策
**特征**: 较低影响、易改变
**示例**:
- 具体算法选择
- 第三方库选择
- 代码组织方式

**记录重点**:
- 决策内容
- 选择理由
- 备选方案
```

### 业务决策
```markdown
**定义**: 业务相关的决策
**特征**: 业务影响、需确认
**示例**:
- 业务规则确定
- 优先级排序
- 范围确定

**记录重点**:
- 决策内容
- 业务依据
- 干系人确认
```

## 阶段 3：迭代中的决策管理

### 3.1 迭代决策清单

```markdown
## 迭代决策清单

### 迭代 [N] 决策
| # | 决策ID | 决策内容 | 类型 | 状态 | 优先级 |

### 决策优先级
```markdown
**P0**: 必须记录，影响架构或核心功能
**P1**: 建议记录，影响实现细节
**P2**: 可选记录，方便后续回溯
```
```

### 3.2 决策变更追踪

```markdown
## 决策变更追踪

### 变更记录
| # | 决策ID | 变更类型 | 变更内容 | 日期 | 原因 |
|---|---------|----------|----------|------|------|
| 1 | DEC-001 | 修改 | [变更] | [日期] | [原因] |
| 2 | DEC-001 | 撤销 | - | [日期] | [原因] |

### 变更类型
```markdown
**修改**: 部分内容变更，决策仍有效
**撤销**: 决策被撤销，不再适用
**替换**: 被新决策替代
```
```

### 3.3 决策状态

```markdown
## 决策状态

### 状态定义
```markdown
**有效**: 决策当前有效，应被遵守
**待验证**: 决策已做但需验证效果
**已变更**: 部分内容被修改
**已撤销**: 决策不再有效
**已实现**: 决策已被实现
```

### 状态转移
```markdown
有效 → 待验证 → 有效/已变更/已撤销
有效 → 已实现
有效 → 已撤销
```
```

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `current_iteration` | 当前迭代 | `3` |
| `decisions_made` | 已做决策 | `[{id: "DEC-3-001", content: "..."}]` |
| `context_history` | 上下文历史 | `[{iteration: 1, context: "..."}]` |

# Usage Notes

1. **记录依据**：不只是记录结论，更重要的是记录依据
2. **替代方案**：记录考虑过的替代方案和未选择原因
3. **影响范围**：明确标注决策的影响范围
4. **状态追踪**：追踪决策的状态变化
5. **可追溯**：建立代码和决策之间的追溯映射

# Example Input

```yaml
current_iteration: 2
decisions_made:
  - id: "DEC-2-001"
    content: "采用Redis作为缓存"
    rationale: ["性能好", "团队熟悉"]
    alternatives_considered:
      - name: "Memcached"
        reason: "未选择原因"
  - id: "DEC-2-002"
    content: "使用JWT认证"
    rationale: ["无状态"]
context_history:
  - iteration: 1
    context: "讨论了缓存方案"
```
