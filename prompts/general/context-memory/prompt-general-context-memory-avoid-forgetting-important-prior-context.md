# Context

在长对话中，某些上下文会因为时间间隔、话题切换或上下文窗口限制而被遗忘。AI 需要主动识别重要上下文并建立保留机制，确保关键信息不会丢失。本 prompt 的核心目标是：**指导 AI 识别和保留重要的先前上下文，建立防止遗忘的机制**。

# Prompt Body

## 阶段 1：重要性识别

### 1.1 重要性标准

```markdown
## 重要性标准

### 高重要性标准
| # | 标准 | 说明 | 示例 |
|---|------|------|------|
| 1 | 约束声明 | 用户明确的要求或限制 | "API不能改" |
| 2 | 决策点 | 影响后续方向的决策 | "选择方案A" |
| 3 | 承诺 | 对用户的承诺 | "下周一完成" |
| 4 | 前提条件 | 任务执行的前置 | "需要先获取数据" |
| 5 | 外部依赖 | 依赖外部系统/团队 | "等支付接口" |

### 中重要性标准
| # | 标准 | 说明 | 示例 |

## 阶段 3：保留机制

### 3.1 保留策略

```markdown
## 保留策略

### 主动保留
```markdown
**时机**: 发现重要上下文时立即保留
**方式**: 
- 在当前输出中引用
- 更新状态摘要
- 标记为重要

**保留格式**:
```markdown
【重要上下文 - 需要保留】
[内容]
原因: [为什么重要]
保留至: [条件/时间]
```

## 阶段 4：遗忘预防

### 4.1 预防机制

```markdown
## 遗忘预防

### 检查点
```markdown
**每次继续任务前检查**:
1. 当前任务的目标是什么？
2. 有哪些必须满足的约束？
3. 之前做了哪些决策？
4. 有什么悬而未决的问题？

**如果发现遗忘**:
1. 在历史中查找
2. 恢复上下文
3. 更新状态摘要
4. 继续任务
```
```

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `conversation_history` | 对话历史 | `[{role: "user", content: "..."}]` |
| `current_context` | 当前上下文 | `{task: "进行中", phase: "执行"}` |
| `importance_criteria` | 重要性标准 | `["约束声明", "决策点"]` |

# Usage Notes

1. **主动识别**：不要被动等待，主动识别重要上下文
2. **分级保留**：不同重要性的上下文用不同的保留方式
3. **预防为主**：建立遗忘预防机制，而非事后恢复
4. **检查点机制**：在关键节点检查重要上下文是否保留
5. **透明沟通**：遗忘时主动向用户说明并恢复

# Example Input

```yaml
conversation_history:
  - role: "user"
    content: "帮我写个函数，参数是数字列表，返回最大值"
  - role: "assistant"
    content: "好的，我会写一个返回最大值的函数"
  - role: "user"
    content: "不能用内置的max函数，要自己实现"
  - role: "assistant"
    content: "明白，用循环实现"
current_context:
  task: "实现最大值函数"
  phase: "执行"
```
