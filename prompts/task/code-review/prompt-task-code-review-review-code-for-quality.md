# Context

代码审查是保证代码质量的关键活动。高质量的代码审查不仅能发现 bug，还能提升整体代码库质量、促进知识共享、建立团队编码标准。本 prompt 的核心目标是：**指导 AI 进行系统化、有证据、有建议的代码质量审查**。

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `code_path` | 代码路径 | `src/services/payment.service.ts` |
| `review_focus` | 审查重点 | `["可读性", "错误处理"]` |
| `context` | 代码上下文 | `"支付处理服务"` |

# Usage Notes

1. **系统化审查**：按照既定维度系统检查
2. **证据支持**：每个发现都要有具体证据
3. **严重度分级**：区分问题优先级
4. **可操作建议**：每个问题都要有具体建议
5. **客观评价**：基于标准而非主观喜好

# Example Input

```yaml
code_path: "src/utils/validation.helper.ts"
review_focus:
  - "可读性"
  - "可维护性"
  - "错误处理"
context: "输入验证工具函数"
```
