# Context

代码风险可能来自多个方面：安全性漏洞、稳定性问题、扩展性限制、依赖风险等。未能识别的风险往往会转化为生产问题。本 prompt 的核心目标是：**指导 AI 系统化地识别和评估代码风险**。

# Prompt Body

## 阶段 1：风险分类

### 1.1 风险类型

```markdown
## 风险类型

| 风险类型 | 描述 | 关注点 |
|----------|------|--------|
| 安全性风险 | 安全漏洞和弱点 | 注入、认证、加密 |
| 稳定性风险 | 导致系统崩溃 | 异常、空指针、资源耗尽 |
| 可扩展性风险 | 限制未来扩展 | 硬编码、单体架构 |
| 依赖风险 | 外部依赖问题 | 版本、更新、维护 |
| 运维风险 | 运维困难 | 日志、监控、配置 |
```

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `code_path` | 代码路径 | `src/api/user.controller.ts` |
| `risk_focus` | 风险重点 | `["安全性", "稳定性"]` |
| `context` | 代码上下文 | `"用户认证 API"` |

# Usage Notes

1. **全面识别**：覆盖所有风险类型
2. **准确评估**：基于影响的真实评估
3. **优先排序**：高风险优先处理
4. **可行缓解**：提供可实施的缓解措施
5. **持续监控**：建立风险监控机制

# Example Input

```yaml
code_path: "src/api/payment.controller.ts"
```
