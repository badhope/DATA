# Context

可维护性是软件生命力的关键指标。高可维护性代码易于修改、扩展和调试；低可维护性代码会导致开发速度下降、bug 增加、技术债务累积。本 prompt 的核心目标是：**指导 AI 系统化地评估和改进代码可维护性**。

# Prompt Body

# Usage Notes

1. **系统评估**：按照既定维度系统评估
2. **债务意识**：识别并量化技术债务
3. **现实建议**：基于实际可行性的建议
4. **优先级分明**：区分问题优先级
5. **长期视角**：考虑代码的长期演化
