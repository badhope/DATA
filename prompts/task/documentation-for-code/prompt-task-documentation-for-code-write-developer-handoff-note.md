# Context

开发者交接是知识传递的重要环节。当团队成员变动或项目交接时，完整的交接文档能帮助接手人快速理解工作内容、顺利开展工作。本 prompt 的核心目标是：**指导 AI 为开发者交接编写完整、清晰的说明文档**。

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `project_context` | 项目上下文 | `{name: "用户系统", type: "微服务"}` |
| `completed_work` | 已完成工作 | `[{feature: "用户注册", status: "done"}]` |
| `pending_work` | 待办工作 | `[{feature: "用户头像", priority: "P1"}]` |
| `key_information` | 关键信息 | `[{type: "config", content: "..."}]` |

# Usage Notes

1. **全面梳理**：系统化梳理所有交接内容
2. **提供上下文**：不要假设接手人知道背景
3. **明确风险**：明确标注风险和需要注意的地方
4. **清晰结构**：使用标题和列表提高可读性
5. **交接确认**：包含交接确认机制

# Example Input

```yaml
project_context:
  name: "订单处理系统"
  type: "微服务"
  team: "订单团队"
completed_work:
  - feature: "订单创建"
    status: "completed"
    location: "src/services/order.service.ts"
pending_work:
  - feature: "订单取消"
    priority: "P1"
    due: "2024-01-20"
key_information:
  - type: "config"
    content: "需要配置支付渠道"
```
