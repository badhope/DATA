# Context

未测试的代码区域是潜在的风险点。识别这些区域并评估其风险，可以帮助团队优先安排测试工作，把有限的测试资源用到最需要的地方。本 prompt 的核心目标是：**指导 AI 系统化地识别和评估未测试的高风险区域**。

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `code_path` | 代码路径 | `src/services/` |
| `existing_tests` | 现有测试 | `["tests/services/"]` |
| `coverage_report` | 覆盖率报告 | `{statement: 75%, branch: 60%}` |

# Usage Notes

1. **系统分析**：基于多个维度评估风险
2. **量化风险**：使用风险矩阵量化风险
3. **优先排序**：高风险优先测试
4. **具体建议**：每个高风险区域都要有具体建议
5. **持续追踪**：建立未测试风险的追踪机制

# Example Input

```yaml
code_path: "src/services/"
existing_tests:
  - "tests/services/user.test.ts"
  - "tests/services/order.test.ts"
coverage_report:
  statement: "75%"
  branch: "60%"
  function: "80%"
```
