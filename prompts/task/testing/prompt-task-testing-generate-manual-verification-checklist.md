# Context

有些测试场景无法自动化，必须通过手动验证完成。手动验证清单是确保这些验证被系统化执行的工具。本 prompt 的核心目标是：**指导 AI 生成完整、可操作的手动验证清单**。

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `change_description` | 变更描述 | `"新增用户头像上传功能"` |
| `verification_scope` | 验证范围 | `{types: ["功能", "UI"], priority: "P0"}` |
| `environment` | 验证环境 | `"测试环境 v2.1"` |

# Usage Notes

1. **具体可操作**：每一步都要具体可执行
2. **明确标准**：每个验证点都要有明确的验收标准
3. **可重复**：提供清晰的测试数据，确保可重复验证
4. **完整性**：覆盖所有关键验证点
5. **记录完整**：记录实际结果，便于追踪

# Example Input

```yaml
change_description: "新增用户头像上传功能"
verification_scope:
  types:
    - "功能验证"
    - "UI验证"
    - "性能验证"
  priority: P0
environment: "测试环境 v2.1"
```
