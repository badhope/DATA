# Context

代码变更时，回归测试计划确保变更不会破坏现有功能。一个好的回归测试计划应该聚焦于可能受影响的区域，而不是简单地运行所有测试。本 prompt 的核心目标是：**指导 AI 创建有效的回归测试计划**。

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `change_description` | 变更描述 | `"修改了用户认证逻辑"` |
| `affected_code` | 受影响的代码 | `["src/auth/login.ts"]` |
| `test_suite` | 现有测试套件 | `{unit: [...], integration: [...]}` |

# Usage Notes

1. **聚焦风险**：优先覆盖高风险区域
2. **依赖分析**：不要遗漏间接影响的区域
3. **合理排序**：按照依赖关系排序测试
4. **时间管理**：平衡覆盖率和时间
5. **风险预案**：准备回滚和缓解计划

# Example Input

```yaml
change_description: "重构订单服务，拆分验证逻辑"
affected_code:
  - "src/services/order.service.ts"
  - "src/services/order.validator.ts"
test_suite:
  unit:
    - "tests/order.service.test.ts"
  integration:
    - "tests/order.api.test.ts"
```
