# Context

工程交接是项目周期中的重要环节，特别是在团队变动、项目交接、模块转移等场景下。系统化的交接计划确保知识完整传递，工作顺利延续。本 prompt 的核心目标是：**指导 AI 生成完整、可执行的工程交接计划**。

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `project_context` | 项目上下文 | `{name: "订单系统", type: "微服务"}` |
| `handover_scope` | 交接范围 | `{include: [...], exclude: [...]}` |
| `timeline` | 时间线 | `{start: "2024-01-01", duration: "4周"}` |

# Usage Notes

1. **系统化梳理**：全面梳理需要交接的内容
2. **分阶段实施**：合理安排交接阶段
3. **知识验证**：确保知识真正传递
4. **应急准备**：准备应急联系方式和方案
5. **签字确认**：建立正式的交接确认机制

# Example Input

```yaml
project_context:
  name: "用户中心"
  type: "微服务"
  team: "用户团队"
handover_scope:
  include:
    - "用户服务代码"
    - "用户数据库"
    - "相关文档"
  exclude:
    - "其他团队的模块"
timeline:
  start: "2024-01-15"
  duration: "4周"
```
