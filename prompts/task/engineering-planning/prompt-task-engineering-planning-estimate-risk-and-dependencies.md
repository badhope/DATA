# Context

风险和依赖评估是技术规划的关键环节。系统化的风险评估帮助团队提前识别问题、制定缓解措施、准备应急预案，确保项目顺利推进。本 prompt 的核心目标是：**指导 AI 系统化地评估技术方案的风险和依赖**。

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `technical_proposal` | 技术方案 | `{name: "微服务改造", components: [...]}` |
| `context` | 上下文 | `"将单体应用拆分为微服务"` |
| `team_experience` | 团队经验 | `["有微服务经验", "无容器化经验"]` |

# Usage Notes

1. **系统化识别**：使用分类框架系统化识别风险
2. **量化评估**：尽可能量化风险概率和影响
3. **明确责任**：每个风险有明确的负责人
4. **可操作的缓解措施**：缓解措施要具体可执行
5. **定期复审**：风险评估需要定期更新

# Example Input

```yaml
technical_proposal:
  name: "引入 Redis 缓存"
  components:
```
