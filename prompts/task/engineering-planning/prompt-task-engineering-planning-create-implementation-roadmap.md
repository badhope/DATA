# Context

实施路线图是将功能从概念到落地的系统性规划。它将大型功能拆分为可管理的阶段，每个阶段有明确的里程碑和验收标准。本 prompt 的核心目标是：**指导 AI 创建清晰、可执行的实施路线图**。

# Variables

| 变量 | 说明 | 示例 |
|------|------|------|
| `feature_description` | 功能描述 | `"实现用户画像系统"` |
| `constraints` | 约束条件 | `{deadline: "2个月", budget: "2人"}` |
| `available_resources` | 可用资源 | `["开发2人", "测试1人"]` |

# Usage Notes

1. **阶段合理**：每个阶段应该可独立交付
2. **里程碑明确**：每个里程碑要有明确的交付物和验收标准
3. **风险预案**：识别风险，准备备选方案
4. **资源明确**：明确人员和资源分配
5. **持续更新**：路线图应该持续更新

# Example Input

```yaml
feature_description: "实现电商订单推荐功能"
constraints:
  deadline: "3个月"
  budget: "3人团队"
available_resources:
  - "后端开发2人"
  - "前端开发1人"
  - "算法1人"
```
