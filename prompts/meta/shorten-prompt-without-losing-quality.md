intent: |
  用于在保持 Prompt 效果的前提下，减少冗余内容，使 Prompt 更简洁高效。
  重点是识别可压缩的部分和必须保留的部分。
applicable_models:
  - "*"
required_inputs:
  - original_prompt: string 原始 Prompt
  - quality_requirements: array 必须保持的质量要求（可选）
outputs:
  - shortened_prompt: string 压缩后的 Prompt
  - removed_elements: array 被移除的元素
  - preserved_elements: array 保留的元素
  - quality_impact: string 质量影响评估
steps:
  - id: 1
    name: 分析原始 Prompt
    action: |
      全面分析原始 Prompt：

      **结构分析**：
      - 识别各部分（角色、任务、约束、示例等）
      - 统计各部分长度
      - 标记关键和非关键内容

      **标注每个部分**：
      ```markdown
      | 部分 | 内容摘要 | 长度 | 重要性 | 可压缩性 |
      |------|----------|------|--------|----------|
      | 角色 | "你是专家" | 4字 | 高 | 低 |
      | 任务 | 长描述 | 200字 | 高 | 中 |
      | ... | ... | ... | ... | ... |
      ```

      **重要性评级**：
      - 高：删除会导致输出质量明显下降
      - 中：删除可能有轻微影响
      - 低：删除几乎无影响
    output: 原始分析

  - id: 2
    name: 识别可压缩内容
    action: |
      找出可以压缩的部分：

      **可压缩模式**：

      1. **冗余描述**
      - 重复表达相同意思
      - 过多修饰词
      - 例："非常非常认真仔细地" → "认真"

      2. **显而易见的说明**
      - 不言自明的内容
      - 常识性内容
      - 例："你需要输出文字，不能输出图片"

      3. **过度解释**
      - 解释为什么要这样做
      - 不必要的背景
      - 过多的"因为"

      4. **冗长的示例**
      - 示例占比过大
      - 示例之间有重复
      - 用一句话能说清的示例

      5. **格式化冗余**
      - 过多的空行
      - 重复的分隔符
      - 过多的层级标题

      **压缩优先级**：
      1. 删低重要性的冗余内容
      2. 压缩中等重要性的冗长描述
      3. 精简高重要性的表达方式
    output: 可压缩清单

  - id: 3
    name: 制定压缩策略
    action: |
      对每个可压缩部分制定策略：

      **策略 1：直接删除**
      - 适用：完全冗余的内容
      - 方法：直接删除，不保留

      **策略 2：缩写/符号替代**
      - 适用：常用表述
      - 方法：使用约定俗成的缩写
      - 例："输入" → "输入："（精简）

      **策略 3：合并**
      - 适用：多个相关的短句
      - 方法：合并为一个复合句或列表

      **策略 4：提炼**
      - 适用：长篇解释
      - 方法：提炼核心意思，用短句表达

      **策略 5：结构优化**
      - 适用：格式化内容
      - 方法：减少空行、使用更紧凑的格式

      **禁止压缩的要素**：
      - 核心任务定义
      - 必须的约束条件
      - 关键输入输出格式
      - 示例的核心部分（可精简但不能删核心）
    output: 压缩策略

  - id: 4
    name: 执行压缩
    action: |
      按策略逐个执行：

      **执行顺序**：
      1. 删除完全冗余的内容
      2. 合并相关短句
      3. 精简长句表述
      4. 优化格式空行
      5. 检查整体长度

      **压缩技巧**：
      - "你应该" → 删除（隐含在任务中）
      - "非常重要的一点是" → 删除
      - 过多的"请确保" → 精简
      - 重复强调 → 保留一次

      **示例**：
      ```
      原文：
      "你需要非常认真地检查每一个细节，确保输出的内容准确无误，
      因为准确是非常重要的，任何错误都会导致严重的后果。"

      压缩后：
      "检查输出的准确性，确保无误。"
      ```
    output: 压缩结果

  - id: 5
    name: 质量验证
    action: |
      验证压缩后的质量：

      **必须满足的要求**：
      1. 核心任务是否仍然清晰？
      2. 关键约束是否都保留了？
      3. 输出格式要求是否完整？
      4. 示例的核心信息是否保留？

      **对比检查**：
      ```markdown
      | 要素 | 原 Prompt | 压缩后 | 状态 |
      |------|-----------|--------|------|
      | 角色 | ✅ | ✅ | 保留 |
      | 任务 | ✅ | ✅ | 保留 |
      | 约束1 | ✅ | ❌ | 丢失！|
      | ... | ... | ... | ... |
      ```

      **功能测试**：
      - 用压缩后的 Prompt 测试
      - 对比输出质量
      - 检查是否有退化

      如有质量问题，回退或调整
    output: 质量验证

  - id: 6
    name: 输出最终结果
    action: |
      最终输出：

      ```markdown
      ## 压缩后的 Prompt

      [完整文本]

      ## 压缩统计
      - 原长度：XXX 字符
      - 压缩后：XXX 字符
      - 减少：XX%（XX 字符）

      ## 移除内容
      - [移除项1]
      - [移除项2]

      ## 保留内容
      - [保留项1]
      - [保留项2]

      ## 质量影响
      - 无影响 / 轻微影响 / 需要注意

      ## 建议
      [如有时]
      ```
    output: 最终结果

anti_patterns: |
  - 不要删除核心任务定义
  - 不要删除关键的约束条件
  - 不要为了压缩而压缩，导致语义不清
  - 不要删除所有的"请"、"确保"等礼貌用语（适度保留）
  - 不要压缩掉所有示例（保留核心示例）

failure_modes: |
  - **过度压缩**：删除了重要内容 → 回退并重新评估重要性
  - **语义改变**：压缩后意思改变 → 对比原文检查语义
  - **可读性下降**：太紧凑难以阅读 → 保持适度格式化

self_check: |
  - [ ] 核心任务是否仍然明确？
  - [ ] 关键约束是否都保留了？
  - [ ] 输出格式要求是否完整？
  - [ ] 压缩后是否仍然可读？
  - [ ] 质量是否有明显下降？
